## 📊 Output Files Explained

### 1. `ast.json`
- Generated by: Parser (Phase 2)
- Contains: Raw Abstract Syntax Tree
- Format: JSON
- Purpose: Represents program structure

### 2. `semantic/annotated_ast.json`
- Generated by: Semantic Analyzer (Phase 3)
- Contains: AST with type annotations
- Additional fields: `dataType`, `semanticInfo`
- Purpose: Type-checked AST

### 3. `semantic/symbol_table.json`
- Generated by: Semantic Analyzer (Phase 3)
- Contains: All declared variables with types and dimensions
- Purpose: Variable tracking and type information

### 4. `codegen/ir.txt`
- Generated by: Code Generator (Phase 4)
- Contains: Three-Address Code (TAC)
- Format: Human-readable intermediate representation
- Purpose: Ready for optimization or target code generation

## 🐛 Debugging Tips

### If Lexer Fails
- Check for unsupported characters
- Verify string/char literals are properly quoted
- Ensure comments are properly closed

### If Parser Fails
- Check syntax against grammar in `CFG.md`
- Verify all brackets/braces are balanced
- Ensure semicolon at end of statement

### If Semantic Analysis Fails
- Check for undeclared variables
- Verify type compatibility
- Ensure array dimensions match initialization

### If Code Generation Fails
- Verify semantic analysis passed
- Check that all files exist
- Ensure symbol table is populated

## 🎓 Understanding the Output

### Lexer Output
Shows how source code is broken into tokens. Each token has:
- **Type**: Category (DATATYPE, IDENTIFIER, etc.)
- **Lexeme**: Actual text value

### Parser Output
Shows hierarchical structure of the program. Key node types:
- **Program**: Root node
- **DeclAssign**: Declaration with initialization
- **Array**: Array literal
- **ArrayRow**: One row of array data

### Semantic Output
Adds type information to every node:
- **dataType**: Resolved type (int, float, etc.)
- **semanticInfo**: Additional semantic data

### Codegen Output
Shows low-level operations:
- **DECL**: Variable declaration
- **ASSIGN**: Simple assignment
- **LOAD**: Read from array
- **STORE**: Write to array
- **Arithmetic**: +, -, *, /

## 📞 Troubleshooting

**Problem:** Semantic phase skipped
**Solution:** Compile semantic analyzer:
```bash
cd semantic
g++ -std=c++17 semantic_main.cpp ast_builder.cpp semantic_analyzer.cpp semantic_output.cpp symbol_table.cpp -o semantic_main
```

**Problem:** Codegen phase skipped
**Solution:** Compile code generator:
```bash
cd codegen
make
```

**Problem:** Compilation errors
**Solution:** Ensure C++17 support:
```bash
g++ --version  # Should be GCC 7+ or Clang 5+
```
